# 数式対応ホームページ制作プロジェクト概要

## 1. プロジェクト要件

- オリジナルのホームページを作成する。
- **【必須要件】数式を美しく表示できること。**

## 2. 推奨技術スタック

モダンで高速、かつメンテナンス性に優れた技術スタックを選定しました。

| カテゴリ          | 推奨技術                | 目的・理由                                                                   |
| :---------------- | :---------------------- | :--------------------------------------------------------------------------- |
| **フレームワーク** | **Next.js (App Router)** | 高速な静的サイト生成(SSG)、優れた開発体験、SEO対策を実現します。             |
| **コンテンツ管理** | **Markdown ファイル** | Gitによるバージョン管理が容易で、エンジニアにとって最も馴染みやすい形式です。    |
| **数式ライブラリ** | **KaTeX** | 高速な数式レンダリングを実現します。静的サイトとの相性が抜群です。             |
| **スタイリング** | **Tailwind CSS** | ユーティリティファーストのアプローチで、開発速度の向上とデザインの一貫性を確保します。 |

---

## 3. 実装計画

### 3.1. 基本方針

- ブログ記事などのコンテンツは、ローカルのMarkdownファイル (`/contents`ディレクトリなど) で管理します。
- Markdownファイル内にLaTeX形式で記述された数式を、サイトのビルド時にKaTeXライブラリを使って美しいHTMLに変換します。

### 3.2. 実装手順の概要

1.  **Next.jsプロジェクトのセットアップ**: `create-next-app` で雛形を作成します。
2.  **ライブラリの導入**: Markdownと数式を処理するため、`gray-matter`, `remark`, `remark-math`, `rehype-katex` 等をインストールします。
3.  **KaTeX用CSSの読込**: 全ページ共通のレイアウトファイル(`app/layout.tsx`)でKaTeX用のCSSをインポートし、スタイルを適用します。
4.  **Markdown処理ロジックの実装**: Markdownファイルを読み込み、数式部分を含めてHTMLに変換する専用の関数を作成します (`lib/posts.ts`など)。
5.  **記事ページの作成**: Next.jsの動的ルーティング (`app/blog/[slug]/page.tsx`) を利用し、生成したHTMLをページに表示します。

### 3.3. 数式の記述方法

Markdownファイル内で、インライン数式は `$` で、ブロックレベルの数式は `$$` で囲んで記述します。

**記述例:**

```markdown
---
title: '私の最初の投稿'
date: '2025-06-28'
---

これはインライン数式の例です: `$E=mc^2$`

以下はブロックレベルの数式の例です:

$$\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}$$
```

4.2. 推奨デプロイ構成
Firebase Hosting を第一の推奨選択肢とします。Vercelに近い手軽な開発体験と、GCPネイティブの信頼性・パフォーマンスを両立できるため、今回のプロジェクトに最適です。

4.3. CI/CDパイプラインの構築
どのデプロイ先を選ぶ場合でも、Cloud Build または GitHub Actions を利用して、Gitリポジトリへのpushをトリガーとした自動ビルド＆デプロイの環境を構築することを強く推奨します。これにより、コンテンツの更新作業が大幅に効率化されます。
